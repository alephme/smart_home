智能家居监控平台（Qt Widgets）项目介绍

本文档基于当前工程源码整理，覆盖整体架构、数据库结构组织、六大模块拆分、关键业务流程和后续可扩展方向。 项目采用 Qt 5 + C++14 + SQLite，以桌面端可视化方式实现智能家居的登录管理、设备控制、场景编排、历史追踪、异常报警与系统配置。

1. 项目定位与技术栈

  • 定位：企业实训型智能家居管理平台，强调“可演示 + 可扩展 + 端到端闭环”。
  • 框架：Qt Widgets（界面） + Qt SQL（数据访问） + Qt Network（设备配对探测）。
  • 数据库：SQLite（默认 smart_home.db，由 DatabaseManager 单例统一管理）。
  • 构建：qmake + MinGW，产物输出到 bin/。
  • 运行模型：本地单进程桌面应用，界面层通过信号槽触发业务操作并写入数据库。

2. 模块总览

登录与首页模块
设备控制与状态监控模块
场景模式设置模块
历史操作与数据查询模块
异常报警模块
系统与设备管理模块

页面容器为 MainWindow + QStackedWidget，登录成功后切换到主导航界面。

3. 架构组织（分层视角）

UI 层（Widget） LoginWidget / HomeWidget / DeviceControlWidget / SceneWidget / HistoryWidget / AlarmWidget / SettingsWidget | | 调用 v 数据访问层（DatabaseManager 单例） users / devices / scenes / scene_devices / operation_logs / env_data / alarms / settings | | 落地 v SQLite（smart_home.db）
  • UI 组件职责清晰，每个模块是独立 Widget，便于替换或升级。
  • DatabaseManager 作为统一数据入口，屏蔽 SQL 细节，供所有模块复用。
  • 主窗口负责页面导航、用户会话状态（当前用户名）和跨模块信号连接。

4. 数据库结构组织（核心）

4.1 表结构

表名 核心字段 用途 关联关系
users id, username, password(sha256), role, created_at 登录认证、账号管理（含注册/重置） 通过 username 参与操作日志记录
devices id, name, type, grp, ip, port, status, params 设备主数据与实时状态 与 scene_devices.device_id 逻辑关联
scenes id, name, description, created_at 场景定义（回家/睡眠/离家等） 与 scene_devices.scene_id 逻辑关联
scene_devices id, scene_id, device_id, action, params 场景与设备动作绑定 连接 scenes 与 devices（多对多桥表）
operation_logs id, username, device_name, action, result, created_at 操作审计、历史追溯、CSV 导出来源 记录用户对设备的行为结果
env_data id, temperature, humidity, air_quality, created_at 环境传感数据存档 用于首页统计与报警阈值判断
alarms id, type, content, device_name, is_read, created_at 异常告警记录与展示 可由环境阈值、设备离线等触发
settings key, value 系统参数持久化（主题、刷新、排序等） 被设备页、系统设置页读取与写入

4.2 数据初始化策略

  • 首次启动自动建表并写入默认管理员账号：admin / admin123（密码存储为 SHA-256）。
  • 若设备表为空，自动插入演示设备；若环境表为空，插入演示环境样本。
  • 自动创建示例场景：回家模式、睡眠模式、离家模式，并在启动时重置演示绑定关系。

4.3 关系与一致性说明

  • 已具备 通过业务方法保持桥表一致性（如场景删除时清理绑定；启动时清理孤立绑定）。
  • 可增强 当前 scene_devices 未声明显式外键约束，后续可补充 FK + ON DELETE CASCADE，提升数据库级一致性。

5. 六大模块详细组织

5.1 登录与首页模块

  • 登录页 LoginWidget：提供登录、注册、密码重置；输入校验后调用 validateUser / addUser / resetPassword。
  • 登录成功发送 loginSuccess(username)，主窗口切换到业务主界面。
  • 首页 HomeWidget：展示在线/离线设备数、报警数量、当前时间；每 5 秒自动刷新。
  • 首页内置快捷入口按钮，直接导航到设备、场景、历史、报警、设置页面。

5.2 设备控制与状态监控模块

  • 设备页 DeviceControlWidget 以卡片方式展示设备，支持类型过滤、添加、编辑、删除、刷新。
  • 支持设备卡片拖拽重排，顺序通过 settings.device_card_order 持久化。
  • 离线设备点击后可尝试 TCP 配对（QTcpSocket connectToHost），成功后自动置为在线。
  • 在线设备可打开控制弹窗，根据设备类型动态渲染控制项： 灯光（亮度/颜色）、空调（模式/温度/风力）、窗帘（开合度）、摄像头（监控开关）。
  • 每次控制结果写入 operation_logs，并触发 deviceChanged 更新首页统计。

5.3 场景模式设置模块

  • 场景页 SceneWidget 管理场景 CRUD，并维护“场景-设备动作”绑定。
  • 新建/编辑时可按设备类型选择预设动作模板，模板包含 action + params。
  • 激活场景时逐设备执行：在线设备更新参数并记日志，离线设备跳过并记 offline_skip。
  • 场景参数与设备当前参数进行 merge，保证未覆盖字段可延续现值。

5.4 历史操作与数据查询模块

  • 历史页 HistoryWidget 由双标签页组成：操作记录、环境数据。
  • 支持按日期范围和设备类型查询，调用 getOperationLogs / getEnvData。
  • 支持 CSV 导出，便于归档、分析或课设汇报。

5.5 异常报警模块

  • 报警页 AlarmWidget 支持阈值配置（温度、湿度）与 10 秒轮询检查。
  • 检测逻辑包括：环境数据超阈值、设备离线状态；报警写入 alarms 并更新表格。
  • 支持删除选中和清空全部；对离线报警有“避免重复”判定。
  • 主窗口可接收报警触发信号并统一追加系统异常记录。

5.6 系统与设备管理模块

  • 系统设置页 SettingsWidget 负责主题、刷新间隔、数据库路径等参数维护。
  • 设备管理能力分布在设备页（设备档案维护）与数据库层（设备状态写入）两部分。
  • 系统配置统一保存到 settings，实现跨次启动记忆。

6. 页面与信号槽协同

  • MainWindow 承担导航条与页面栈管理，页面索引固定：主页、设备、场景、历史、报警、设置。
  • HomeWidget::navigateTo 直接驱动页面跳转，形成“总览 -> 业务页”快速流。
  • DeviceControlWidget::deviceChanged 连接到首页刷新，保证统计一致。
  • AlarmWidget::alarmTriggered 连接主窗口后，能够统一补充“系统异常”告警记录。

7. 典型业务流程

用户登录 -> MainWindow 构建导航与页面栈 -> 设备页控制设备（更新 devices + 记录 operation_logs） -> 场景页激活场景（批量更新 devices + 记录 operation_logs） -> 报警页定时检查（写入 alarms） -> 历史页查询/导出（读取 operation_logs 与 env_data）

8. 当前工程亮点与可优化点

亮点

  • 模块边界清晰，UI 与数据访问解耦，适合教学与二次开发。
  • 数据链路完整：控制动作、场景执行、报警生成、历史追溯形成闭环。
  • 演示数据初始化完善，开箱即可运行展示。

可优化点

  • 可将数据库查询字符串拼接改为参数化，进一步提升安全性与可维护性。
  • 可引入真正的设备协议层（MQTT/Modbus/HTTP API），替代当前演示级 TCP 连通校验。
  • 可补充权限分级（admin/user）在页面级、操作级的限制逻辑。
  • 可加入自动化测试（数据库接口测试、关键业务流程测试、UI 冒烟测试）。
